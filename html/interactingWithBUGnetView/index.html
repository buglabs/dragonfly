Interacting with the BUGnet View

The BUGnet view allows you to interact with the BUG community. You can browse, upload, or download Applications on http://www.buglabs.net.

You can:

  • left click on any Application to bring up the BUGnet Application details page in a Browser window. From this page you can click to Download Latest Version to SDK link to download the Application.
  • right click on any Application and select Import to Dragonfly SDK to download the Application.
  • click the refresh button to refresh the BUGnet view.
  • click the Search Applications button to bring up BUGnet Applications page to use the search functions.
  • right or left click on any section toggle expand/collapse for the section.

If you accidently close this view you can go to "Window > Show View > Other > Dragonfly > BUgnet" to display again.

BUGnet Preferences

A number of settings for the BUGnet view can be accessed via the Eclipse Preferences. Go to: Window > Preferences > BUGnet
  • Enable BUGnet - checkbox that allows you to go into offline mode

  • Server Name - The web service URL to buglabs.net.

  • Number of Applications to display - Accepts 0 to 100. This changes the number of Applications displayed in the My Apps and Top Apps sections.

  • Clear Authentication Data - Clears saved BUGnet Authentication and logs you out of your current session.

My Apps

This section displays any Applications you have uploaded to BUGnet or any that you have been invited to collaborate on by another BUG user.

Click the log-in link to bring up the BUGnet Authentication window.



You can enter an existing BUGnet Username and Password and click Ok (Optional: check the Save Authentication checkbox). If you don't have BUGnet account you can click the here link to create one.

Latest Apps

This section displays the latest updated Applications on BUGnet.

Applications by Modules

This section displays when a Physical Editor is open for any BUG connection (see: MyBUGs view). It returns a list of Applications based on the Modules attached to the BUG. When user has multiple BUG connections this section displays for whichever Physical Editor was last active.
